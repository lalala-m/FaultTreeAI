# FaultTreeAI 项目开发路线图

> 更新时间：2026-04-07
> LangChain 架构重构计划 - **全部完成**

---

## 一、重构完成总结

### 架构变化

| 模块 | 旧实现 | 新实现 |
|------|--------|--------|
| **Embeddings** | 3处重复实现 | `backend/core/llm/embeddings.py` 统一实现 |
| **ChatModel** | 非标准 Provider | `MiniMaxChatModel`, `OllamaChatModel` 继承 `BaseChatModel` |
| **LLM 管理** | `LLMManager` | `ProviderFactory` + `LLMManager`(兼容) |
| **检索链** | 手写流程 | LCEL Chain |
| **API** | 直接依赖 | 双模式支持（Chain + 兼容） |

## 二、新架构目录

```
backend/core/
├── llm/
│   ├── embeddings.py          # 统一 Embedding（minimax, ollama, openai, azure_openai）
│   ├── llm_client.py         # ChatModel（MiniMax, Ollama）
│   ├── manager.py            # ProviderFactory + LLMManager
│   ├── base_provider.py      # 基础数据模型
│   ├── structured_generator.py # 保留（向后兼容）
│   └── providers/            # 保留（向后兼容）
├── rag/
│   ├── __init__.py
│   ├── hybrid_retriever.py    # 混合检索（向量 + BM25）
│   ├── pgvector_retriever.py # 向量检索
│   └── bm25_retriever.py     # BM25 检索
├── langchain/
│   ├── chains/
│   │   ├── fault_tree_chain.py  # 故障树生成 LCEL 链
│   │   └── retrieval_chain.py    # RAG 检索链
│   ├── prompts/
│   │   └── fault_tree_prompts.py # Prompt 模板
│   └── output_parsers/
│       └── fault_tree_parser.py   # JSON 解析器
└── api/
    └── generate.py           # 双模式生成
```

## 五、ProviderFactory 使用指南

```python
from backend.core.llm.manager import ProviderFactory

# 获取 ChatModel
chat = ProviderFactory.get_chat_model("minimax")  # 或 "ollama", "openai", "azure_openai"

# 获取 Embeddings
embed = ProviderFactory.get_embeddings("minimax")  # 或 "ollama", "openai", "azure_openai"

# 列出支持的 Provider
print(ProviderFactory.list_chat_providers())   # ['minimax', 'ollama', 'openai', 'azure_openai']
print(ProviderFactory.list_embed_providers())  # ['minimax', 'ollama', 'openai', 'azure_openai']

# 向后兼容
from backend.core.llm.manager import get_llm_manager
manager = get_llm_manager()
```
