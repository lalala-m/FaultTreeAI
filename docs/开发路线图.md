# FaultTreeAI 开发路线图

> 创建时间：2026-03-21
> 当前版本：v3.0（MVP已完成）
> 竞赛目标：基于Llama 3的工业设备故障树智能生成系统

---

## 一、项目背景（竞赛版）

### 1.1 赛题核心要求

| 指标 | 要求 | 说明 |
|------|------|------|
| 故障树结构准确率 | ≥80% | AI生成的故障树结构符合FTA规范 |
| 关联关系提取准确率 | ≥85% | 顶事件-中间事件-底事件关系正确 |
| 硬件环境 | RTX 4050 6G | 需轻量化部署 |
| 部署方式 | 全本地私有化 | 无外网、无付费API |

### 1.2 技术选型

```
基础模型：Meta Llama 3 8B Instruct
量化格式：Q4_K_M（GGUF）
推理引擎：Ollama 0.1.40+
显存占用：≤5.2GB（适配6G显存）
```

## 三、开发阶段规划

### 阶段一：环境搭建（Day 1-2）

**目标：** 本地部署 Llama 3 + Ollama

**任务清单：**

| 序号 | 任务 | 产出 |
|------|------|------|
| 1.1 | 安装 Ollama Windows 版 | 推理引擎就绪 |
| 1.2 | 拉取 Llama 3 8B 量化模型 | `ollama pull llama3:8b` |
| 1.3 | 验证模型响应 | 测试故障推理Prompt |
| 1.4 | 安装 Python 依赖 | requirements.txt |

**验证标准：**
- 模型加载成功，显存占用 ≤5.2GB
- 响应速度 10-20 字/秒
- 能正确解析JSON输出

---

### 阶段二：核心算法模块（Day 3-5）

**目标：** 故障树计算引擎

**任务清单：**

| 序号 | 任务 | 文件 |
|------|------|------|
| 2.1 | 图结构构建 | `backend/core/fta/builder.py` |
| 2.2 | MOCUS最小割集算法 | `backend/core/fta/mocus.py` |
| 2.3 | Birnbaum重要度计算 | `backend/core/fta/importance.py` |
| 2.4 | 逻辑校验（三层校验） | `backend/core/validator/checker.py` |

**算法要求：**
```
1. 循环依赖检测 - NetworkX simple_cycles
2. 孤立节点检测 - 图连通性分析
3. 逻辑门输入校验 - AND门≥2输入，OR门≥2输入
4. 底事件子节点检测 - 基本事件不应有下游
```

**任务清单：**

**Prompt 核心结构：**

## 四、核心Prompt设计（竞赛核心）

### 4.1 故障树生成Prompt

```python
FAULT_TREE_SYSTEM_PROMPT = """你是一个专业的工业设备故障树分析（FTA）专家。

## 任务
根据故障描述和设备知识，生成符合FTA规范的结构化故障树。

## 输出格式（严格JSON）
{
  "top_event": "顶事件描述",
  "confidence": 0.95,
  "analysis_summary": "分析摘要",
  "nodes": [
    {"id": "N1", "type": "top", "name": "顶事件", "description": ""},
    {"id": "N2", "type": "intermediate", "name": "中间事件", "description": ""},
    {"id": "N3", "type": "basic", "name": "底事件1", "description": ""},
    {"id": "N4", "type": "basic", "name": "底事件2", "description": ""}
  ],
  "gates": [
    {"id": "G1", "type": "OR", "output_node": "N2", "input_nodes": ["N3", "N4"]},
    {"id": "G2", "type": "AND", "output_node": "N1", "input_nodes": ["N2"]}
  ]
}

## 规范要求
1. type可选值：top(顶事件), intermediate(中间事件), basic(底事件)
2. gate type可选值：AND(与门), OR(或门)
3. 底事件(type=basic)不能有子节点
4. 逻辑门至少2个输入
5. 禁止循环依赖
6. confidence为0-1之间的置信度

## 设备知识
{context}

## 故障描述
{top_event}

请生成故障树JSON："""
