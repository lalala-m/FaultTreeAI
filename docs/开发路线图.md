## 三、多模型 Provider 架构

### 3.1 核心组件

```
backend/core/llm/
├── __init__.py                    # 导出 LLMManager 单例
├── base_provider.py               # 抽象基类 BaseLLMProvider
│                                  #   + BenchmarkResult 数据类
├── manager.py                     # LLMManager 核心
│                                  #   - 注册多 Provider
│                                  #   - generate_with_fallback()
│                                  #   - benchmark_providers()
│                                  #   - 全局单例 get_llm_manager()
├── providers/
│   ├── __init__.py
│   ├── ollama_provider.py         # Ollama 本地实现
│   └── minimax_provider.py       # MiniMax 云端实现
└── structured_generator.py       # 已改用 manager（自动回退）
```

### 3.2 配置项（`.env` / `.env.docker`）

```bash
# ── 主 Provider（Ollama 本地，零成本）──────────────
LLM_PROVIDER=ollama

# ── 回退 Provider（MiniMax 云端）─────────────────
LLM_FALLBACK_PROVIDER=minimax

# ── Ollama 配置 ──────────────────────────────────
OLLAMA_BASE_URL=http://localhost:11434
OLLAMA_MODEL=qwen2.5:14b-instruct
OLLAMA_EMBED_MODEL=nomic-embed-text

# ── MiniMax 配置 ─────────────────────────────────
MINIMAX_API_KEY=your_api_key
MINIMAX_GROUP_ID=your_group_id
MINIMAX_MODEL=MiniMax-M2
MINIMAX_BASE_URL=https://api.minimaxi.com

# ── Embedding（使用 Ollama 本地，零成本）──────────
EMBED_PROVIDER=ollama
EMBED_DIM=768
```

### 3.3 自动回退逻辑

```
generate_with_fallback(prompt)
    │
    ├──→ primary (LLM_PROVIDER) 尝试
    │       └── 成功 → 返回 (response, provider_name)
    │       └── 失败 → 记录错误，自动切换
    │
    └──→ fallback (LLM_FALLBACK_PROVIDER) 尝试
            └── 成功 → 返回 (response, provider_name)
            └── 失败 → 抛出 RuntimeError（含两条错误信息）
```

**响应字段新增 `provider`**：API 返回的 `GenerateResponse` 中包含本次实际使用的模型名称，方便追踪。

### 4.1 运行方式

### 4.2 对比指标

| 指标 | 说明 | 权重 |
|------|------|------|
| JSON 可解析率 | LLM 输出能被正确解析为 JSON 的比例 | 40% |
| 字段完整度 | `nodes`/`gates`/`confidence`/`analysis_summary` 完整程度 | 40% |
| 平均延迟 | 端到端响应时间（ms） | 20% |

### 4.3 评分算法

```
综合得分 = JSON可解析率×0.4 + 字段完整度×0.4 + 延迟得分×0.2
延迟得分 = 1 - min(avg_latency_ms / 30000, 1.0)   # 30s 封顶
```

## 九、FAQ

### Q1: 如何选择 Ollama 模型？

推荐按推理能力排序：

| 模型 | 参数量 | 内存需求 | 推荐场景 |
|------|--------|----------|----------|
| `qwen2.5:14b-instruct` | 14B | ~10GB | 日常开发首选 |
| `mistral:7b-instruct` | 7B | ~5GB | 低配机器 |
| `deepseek-r1:14b` | 14B | ~10GB | 复杂推理任务 |
| `llama3.1:8b` | 8B | ~6GB | 通用场景 |

下载命令：
```bash
ollama pull qwen2.5:14b-instruct
ollama pull nomic-embed-text    # 必须，用于 RAG 向量化
```

### Q2: Ollama 和 MiniMax 如何选择？

| 场景 | 推荐 | 原因 |
|------|------|------|
| 日常开发调试 | Ollama | 零成本、无限次、离线可用 |
| 正式演示 / 竞赛 | MiniMax | 推理能力强、响应快 |
| 网络不稳定 | Ollama | 完全本地，无网络依赖 |

当前默认配置：**Ollama 主用，MiniMax 自动回退**。

### Q3: RTX 4050 够用吗？

- **Ollama 推理**：不依赖 GPU，使用 CPU 运行（14B 模型约需 10GB 内存）
- **Embedding**：Ollama 本地计算，无需 GPU
- **4050 用途**：如果需要本地加速推理，可开启 GPU 支持：
  ```bash
  ollama pull qwen2.5:14b-instruct
  # Ollama 自动检测并使用 NVIDIA GPU
  ```

### Q4: 如何扩展接入新模型？

新增 Provider，只需实现 `BaseLLMProvider` 接口：

```python
# 1. 创建 provider 文件
# backend/core/llm/providers/openai_provider.py

from backend.core.llm.base_provider import BaseLLMProvider

class OpenAIProvider(BaseLLMProvider):
    name = "openai"

    async def generate(self, prompt: str, **kwargs) -> LLMResponse:
        # 实现generate逻辑
        ...

    async def embed(self, text: str) -> EmbedResult:
        ...

    async def embed_batch(self, texts: list[str]) -> list[EmbedResult]:
        ...

# 2. 注册到 manager
# backend/core/llm/manager.py

_PROVIDER_REGISTRY = {
    "ollama": OllamaProvider,
    "minimax": MiniMaxProvider,
    "openai": OpenAIProvider,   # 新增
}

# 3. 在 .env 中设置
LLM_PROVIDER=openai
```
