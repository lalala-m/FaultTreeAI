## 三、开发阶段规划

### 阶段一：Prompt工程优化（Day 1-3）

**目标：** 提升故障树生成准确率至 ≥80%

**任务清单：**

| 序号 | 任务 | 文件 |
|------|------|------|
| 1.1 | 设计结构化输出Prompt | `backend/core/llm/prompts.py` |
| 1.2 | 添加Few-shot示例 | Prompt中嵌入3-5个示例 |
| 1.3 | 输出JSON严格校验 | 后端解析+重试机制 |
| 1.4 | 领域知识约束 | 引入FTA规范检查 |

**Prompt 核心结构：**
```python
FAULT_TREE_PROMPT = """
你是一个专业的工业设备故障树分析（FTA）专家。

## 输出格式（严格JSON，禁止其他内容）
{
  "top_event": "顶事件描述",
  "confidence": 0.95,
  "analysis_summary": "分析摘要",
  "nodes": [
    {"id": "N1", "type": "top", "name": "顶事件", "description": ""},
    {"id": "N2", "type": "intermediate", "name": "中间事件", "description": ""},
    {"id": "N3", "type": "basic", "name": "底事件1", "description": ""},
    {"id": "N4", "type": "basic", "name": "底事件2", "description": ""}
  ],
  "gates": [
    {"id": "G1", "type": "OR", "output_node": "N2", "input_nodes": ["N3", "N4"]},
    {"id": "G2", "type": "AND", "output_node": "N1", "input_nodes": ["N2"]}
  ]
}

## FTA规范要求
1. type: top(顶事件), intermediate(中间事件), basic(底事件)
2. gate type: AND(与门), OR(或门)
3. 底事件(basic)不能有子节点
4. 逻辑门至少2个输入
5. 禁止循环依赖

## 示例
输入：电机无法启动
输出：{...}

## 设备知识
{context}

## 故障描述
{top_event}

请生成故障树JSON（仅输出JSON，不要其他内容）：
"""
```

## 七、FAQ

### Q1: MiniMax API 如何获取？
- 访问 MiniMax 开放平台：https://platform.minimax.io/
- 注册账号 → 创建API Key → 充值使用
- 也可通过阿里百炼平台调用

### Q2: 需要外网环境吗？
- 是的，MiniMax API 需要外网访问
- 如需纯本地无外网方案，可额外部署 Ollama + Llama 3

### Q3: RTX 4050 够用吗？
- 推理使用API，不依赖本地GPU
- 向量生成也调用API，本地无需大显存
- 4050 可用于加速其他计算任务
